# **\[개발 명세서\] 로컬 지식 봇 (Security-First Development Detail)**

## **1\. 시스템 디렉토리 구조 (Folder Structure)**

보안 가이드에 따라 인증 정보와 데이터, 코드를 엄격히 분리하여 관리합니다.

/local-knowledge-bot  
├── /documents           \# 인덱싱 대상 원본 문서 (사내망 로컬 보관)  
├── /storage             \# 로컬 벡터 데이터베이스 (HNSWLib 인덱스 파일)  
├── /public              \# 프론트엔드 정적 파일  
│   ├── index.html       \# 메인 채팅 인터페이스  
│   ├── style.css        \# UI 디자인 정의서 기반 스타일링  
│   └── script.js        \# 클라이언트 사이드 로직 (보안상 민감 로직 배제)  
├── server.js            \# Node.js 백엔드 (API Key 및 인증 정보 처리 전담)  
├── .env                 \# \[보안\] Gemini API Key 등 환경 변수 (Git/공유 금지)  
├── .gitignore           \# \[보안\] .env 및 /storage 제외 설정  
└── package.json         \# 의존성 라이브러리 목록

## **2\. 보안 강화 데이터 흐름 (Security Data Flow)**

### **A. 문서 전처리 및 인덱싱 (Ingestion Phase)**

1. **로컬 로드**: fs 모듈로 /documents 폴더 스캔. (외부 서버 전송 절대 금지)  
2. **텍스트 추출**: pdf-parse 등을 통해 텍스트화 진행.  
3. **기밀 필터링**: 텍스트 내 개인정보(주민번호, 전화번호 등) 포함 여부를 검토하는 로직 수행.  
4. **로컬 임베딩**: 추출된 텍스트 조각을 벡터화하여 로컬 /storage에 저장.  
   * *보안*: 실제 운영 데이터 대신 테스트 단계에서는 더미(Dummy) 데이터를 사용하여 검증.

### **B. 안전한 질문 및 답변 (Query Phase)**

1. **입력 검증**: 사용자가 입력한 프롬프트에 기밀 정보(계정 정보, 영업 비밀)가 포함되었는지 1차 검증.  
2. **컨텍스트 검색**: 질문과 유사한 문서 조각을 로컬 저장소에서 추출.  
3. **프롬프트 구성**: "아래 제공된 \[문서 내용\]에 기반해 답변하고, \[파일명\]을 출처로 표시해줘"라는 시스템 프롬프트 부여.  
   * *보안*: 프롬프트 구성 시 실제 서버 경로 등 시스템 내부 정보 노출 차단.  
4. **API 호출**: 서버 측에서 .env에 저장된 API Key를 사용하여 Gemini API 호출. (Client-Side 호출 금지)

## **3\. 백엔드 보안 API 명세 (API Endpoints)**

* **POST /api/ingest**: /documents 폴더 파일 인덱싱. (팀장 승인 후 실행 권고)  
* **POST /api/ask**: 질문 수신 및 RAG 수행. API Key는 서버 환경 변수에서 참조.  
* **GET /api/files**: 현재 학습된 파일명 리스트 반환. (실제 파일 경로는 마스킹 처리)

## **4\. 보안 지침 준수 상세 (Security Compliance)**

### **1\) 인증 및 기밀 정보 관리**

* **API Key 격리**: 모든 API Key는 소스코드에 하드코딩하지 않고 .env 파일을 통해서만 관리합니다.  
* **인증 분리**: 개발 및 테스트 과정에서 실제 사내 계정 정보를 AI 대화창이나 코드에 입력하지 않습니다.

### **2\) 파일 및 경로 보안**

* **경로 은닉**: 서버 내부의 실제 디렉토리 구조나 중요 파일 경로는 사용자(프론트엔드)에게 노출하지 않습니다.  
* **업로드 제한**: 파일 업로드 기능 구현 시, 허용된 확장자(PDF, DOCX) 외에는 차단하며 크기를 10MB로 제한하는 필터 로직을 포함합니다.

### **3\) 권한 및 검증**

* **최소 권한**: 로컬 DB(HNSWLib) 및 파일 시스템 접근 시 읽기 전용(Read-only) 권한 위주로 설정합니다.  
* **코드 리뷰**: AI가 생성한 모든 로직은 배포 전 반드시 사람이 보안 취약점(SQL Injection, XSS 등) 유무를 검토합니다.

## **5\. 사용 이력 관리 (Logging)**

* **로컬 로그 기록**: 모든 질문(Prompt)과 답변 내역은 로컬 DB에 기록하여 추후 팀 단위 이력 관리 및 보안 감사에 활용합니다.
