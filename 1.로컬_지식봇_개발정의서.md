# **\[개발정의서\] 로컬 지식 봇 (Local-First RAG PoC)**

## **1\. 제품의 목적 (The "Why")**

* **사내 보안 규정 준수**: 문서 원본과 인덱스(데이터베이스)를 외부 클라우드가 아닌 로컬 PC 내에만 보관하여 정보 유출 원천 차단.
* **업무 효율 극대화**: 흩어져 있는 20\~50개의 HR 규정 및 사내 문서를 AI가 학습하여 10초 이내에 정확한 답변과 근거 출처 제공.

### **2\. 시스템 아키텍처 (The "How")**

* **타입**: 로컬 호스팅형 웹 애플리케이션 (Single PC Hosting)
* **핵심 로직 (RAG)**:
  1. **문서 로드**: 로컬 특정 폴더의 PDF/Word 문서 읽기 (Node.js fs 모듈)
  2. **텍스트 처리**: 문서 내용을 의미 있는 조각으로 분할 (Chunking)
  3. **로컬 임베딩**: 로컬 벡터 저장소(HNSWLib 등)에 인덱싱하여 내 PC에 물리적 저장
  4. **질문/답변**: 사용자 질문 시 관련 조각 검색 \-\> Gemini API에 컨텍스트로 전달 \-\> 답변 생성 및 출처 표시

## **3\. 기술 스택 (The "Tools")**

* **언어**: JavaScript (Runtime: Node.js) / HTML5 / CSS3
* **프레임워크**: Express (Backend API) \+ Vanilla JavaScript (Frontend)
* **AI 엔진**: Google Gemini 1.5 Flash/Pro API (추론 및 요약 담당)
* **문서 처리**: LangChain.js (RAG 파이프라인 구성의 핵심 도구)
* **로컬 DB**: HNSWLib (Node.js 환경에서 별도 서버 없이 파일로 저장 가능한 벡터 엔진)

## **4\. 핵심 기능 (The "What")**

* **Local Doc Scan**: 지정된 로컬 폴더 내의 신규 문서를 자동으로 감지하고 인덱싱.
* **AI Q\&A**: Gemini API를 활용하여 자연스러운 구어체 질문에 답변.
* **Source Citation**: 답변 끝에 "참고: \[파일명\]"을 명시하여 할루시네이션(환각) 방지 및 신뢰도 확보.
* **Custom UI**: 업무 몰입도를 높이는 모던하고 직관적인 웹 채팅 인터페이스.

## **5\. 보안 및 운영 가이드**

* **데이터 격리**: 모든 데이터베이스 파일은 프로젝트 내 storage/ 폴더에만 존재하며 외부 서버로 동기화되지 않음.
* **API 보안**: API Key는 환경 변수(.env)로 관리하여 코드에 노출되지 않도록 설정.
* **배포**: 로컬 PC에서 실행 후, 동일 사내망(LAN) 내에서 IP 주소를 통해 동료들과 테스트 가능.
