# **\[마스터 플랜\] 로컬 PC 기반 사내 지식 봇 구축 가이드**

본 문서는 사내 보안을 유지하면서 Gemini AI의 강력한 능력을 활용하여 HR 및 사내 규정 질의응답 시스템을 구축하기 위한 통합 가이드라인입니다.

## **1\. 프로젝트 개요 및 목적**

* **제품명**: 로컬 보안 지식 봇 (Local-First Knowledge Bot)  
* **핵심 미션**: 사내 문서(PDF/Word)를 외부 클라우드 저장 없이 로컬에서 학습하여, 직원들의 질문에 즉각적이고 정확한 답변과 출처를 제공함.  
* **주요 가치**: 사내 보안 규정 100% 준수, 업무 효율 극대화(질의응답 10초 이내), 정보의 신뢰성 확보.

## **2\. 시스템 아키텍처 (The "How")**

* **작동 원리 (RAG)**:  
  1. **문서 로드**: 로컬 폴더(/documents)의 파일에서 텍스트 추출.  
  2. **청킹(Chunking)**: 의미 있는 단위로 텍스트 분할.  
  3. **임베딩 및 인덱싱**: 로컬 벡터 저장소(HNSWLib)에 물리적 파일로 저장.  
  4. **검색 및 생성**: 질문 발생 시 로컬 DB에서 관련 조각을 찾아 Gemini API에 컨텍스트로 전달하여 답변 생성.

## **3\. 기술 스택 및 개발 환경**

* **런타임**: Node.js (v18 이상 권장)  
* **백엔드**: Express.js  
* **프론트엔드**: HTML5, CSS3 (Tailwind CSS CDN), Vanilla JavaScript  
* **AI 엔진**: Google Gemini 1.5 Pro / Flash (API 기반)  
* **라이브러리**: LangChain.js, HNSWLib-node, pdf-parse, dotenv  
* **개발 환경**: VS Code, Google AI Studio (API Key 발급)

## **4\. 화면(UI/UX) 디자인 정의**

* **컨셉**: 신뢰감을 주는 Professional & Simple 스타일 (Blue/Gray 톤)  
* **구조**:  
  * **좌측 사이드바**: 현재 학습된 문서 리스트 및 로컬 보안 상태 표시.  
  * **우측 메인 채팅**: 카드 스타일의 대화창, 타이핑 애니메이션 효과.  
  * **하단 입력창**: 플로팅 스타일의 질문 입력칸.  
* **특화 기능**: 답변 하단 '참고 문서' 칩(Chip) 표시, 간편 복사 버튼.

## **5\. 상세 개발 명세**

* **폴더 구조**: /documents, /storage, /public, server.js, .env로 구성된 독립적 구조.  
* **API 엔드포인트**:  
  * POST /api/ingest: 로컬 문서 새로고침 및 인덱싱.  
  * POST /api/ask: 사용자 질문에 대한 RAG 기반 답변 생성.  
* **데이터 흐름**: 문서는 로컬에 머물고, '추출된 텍스트 조각'만 답변 생성을 위해 API로 전달됨.

## **6\. 핵심 보안 및 운영 지침**

| 항목 | 등급 | 세부 지침 |
| :---- | :---- | :---- |
| **기밀 최소화** | 필수 | 개인정보, 영업비밀은 입력 금지하며 업무 목적에 한정함. |
| **인증 분리** | 필수 | API Key, 계정 정보는 .env로 관리하며 소스코드 하드코딩 절대 금지. |
| **최소 권한** | 필수 | 로컬 파일 시스템 및 API 권한은 업무에 필요한 최소한으로 부여. |
| **경로 은닉** | 권고 | 서버 내부 디렉토리 구조를 외부(프론트엔드)에 직접 공개하지 않음. |
| **데이터 격리** | 필수 | 모든 인덱스 데이터는 로컬 storage/에만 저장 (Cloud DB 사용 금지). |

## **7\. 프로토타입 생성용 황금 프롬프트**

이 시스템을 구현하기 위해 AI에게 전달할 최종 명령어입니다.

**\[Prompt\]**

너는 Node.js와 LangChain.js 전문가야. 사내 보안을 최우선으로 하는 '로컬 지식 봇'의 프로토타입 코드를 작성해줘.

1. **로컬 RAG 시스템**: /documents 폴더의 PDF 파일을 읽어 HNSWLib 로컬 벡터 스토어에 저장하고 검색하는 로직을 구현해.  
2. **보안 지침**: 모든 API Key는 .env의 GOOGLE\_API\_KEY를 참조하고, 문서는 로컬에서만 텍스트를 추출해.  
3. **백엔드(server.js)**: Express로 /api/ask, /api/ingest를 만들고 시스템 프롬프트에 출처 명시를 강제해줘.  
4. **프론트엔드(index.html)**: Tailwind CSS로 사이드바가 있는 모던한 채팅 UI와 타이핑 효과를 구현해줘.  
5. **제약**: 'DCM' 단어 사용 금지, 상세 주석 포함, 실행 방법 명시.
